# Source Patterns

This file records local skill pattern absorption. It is not a claim that those skills contain official platform truth. Treat every local skill pattern as creative operating knowledge that must be checked against user evidence and current platform sources when the user asks for latest rules.

## Source quality ladder

1. `official platform source`: platform-owned support, newsroom, creator center, transparency center, or policy material.
2. `local skill pattern`: reusable judgment pattern from local skills or Gxgen skill references.
3. `third-party hypothesis`: agency, analytics vendor, media, report, or practitioner analysis.
4. `unsupported lore`: unverified numeric weights, traffic-pool formulas, or secret ranking claims. Do not use as truth.

## absorbed pattern: dbs-hook

Source type: `local skill pattern`

Absorbed pattern:

- A hook is a sample of the content, not an isolated trick.
- Check content material before optimizing the first seconds.
- Strong openings combine topic, hook, and credibility.
- Useful hook raw material includes data, contrast, story, memorable line, authority, and pain.

Applied here:

- `ContentCreationStrategy.scriptPromise` must be fulfillable by the body.
- `proofAsset` must exist before recommending a stronger hook.
- Opening advice should stop when the content has no value or proof to sample.

## absorbed pattern: dbs-xhs-title

Source type: `local skill pattern`

Absorbed pattern:

- Xiaohongshu titles should map to a psychological trigger and a user search/decision state.
- Formula choice matters more than generic title generation.
- Useful triggers include cognitive conflict, curiosity gap, loss avoidance, identity fit, number anchoring, result promise, social proof, controversy, and interaction.

Applied here:

- `audienceIntent` and `contentAngle` must drive cover/title and search guidance.
- Xiaohongshu strategy should preserve save/search value before hard-sell pacing.
- The selected interaction behavior should be explicit: save, compare, comment, or consult.

## absorbed pattern: bggg-tiktok-readvideo

Source type: `local skill pattern`

Absorbed pattern:

- Read TikTok videos through four layers: semantic, visual, rhythm, and business.
- Segment by function: hook, problem, proof, process, comparison, CTA, and B-roll.
- Prefer visual proof over talking-head explanation when both communicate the same claim.

Applied here:

- `proofAsset` separates claims from evidence.
- `editingImplication` must describe which proof scenes support the strategy.
- The strategy should translate analysis into publishing or commercial objective, not only summarize scenes.

## absorbed pattern: ugc-commerce-video-skill

Source type: `local skill pattern`

Absorbed pattern:

- Analyze by commercial function, not scene description alone.
- First 1-3 seconds are the highest-risk zone.
- Platform-native content should feel consumable in-feed while preserving selling clarity.
- Hooks fail when they lack clarity, relevance, tension, curiosity, contrast, or visual readability.

Applied here:

- `ContentCreationStrategy` must precede timeline advice.
- Platform-native guidance should preserve trust and proof, not blindly imitate trends.
- `editingImplication` must connect hook, proof, CTA, captions, and safe-zone choices to platform behavior.
